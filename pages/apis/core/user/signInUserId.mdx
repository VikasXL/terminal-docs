# <MasterChip value={"POST"} /> signInUserId

## Details

| Property | Value | 
| :--- | :--- | 
| Version | `v1` |
| Allowed Connection Types | `rpc`  |
| Allowed Deployment Kinds | `beta` `prod` `localHost` `test`  |
| Allowed Device Types | `test` `ios` `android` `web` `agent`  |
| SLA | time: 600 ms, timeout: 6000 ms |
| Token | refreshOptional |
| Publishes | `callerDeviceList`  |
| Scope | global |


- **Publishes** `callerDeviceList` 
- **Scope** global


## Description

SignIn with user id. This method could be used by support if user loses all the handles. Support
should first verify that the layoutHandle user lost is part of stale handles and give out user id to
sign in. The user subsequently has to add a layoutHandle.

## Message format

```json
{
  "userId": "UserId",
  "password": "string",
  "deviceName": "string",
  "deviceType": "android | ios | web | agent | test",
  "rememberMe": "boolean"
}
```

## Signal format

```json
{
  "refreshToken": "string",
  "callerId": "UserId",
  "callerDeviceId": "DeviceId",
  "clusterToken": "string",
  "rememberMe": "boolean"
}
```
